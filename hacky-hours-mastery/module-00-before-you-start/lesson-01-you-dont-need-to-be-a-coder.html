You don't need to be a coder

If you've never written a line of code, never opened a terminal, and never used Claude Code — you're in exactly the right place. This short module gets you from nothing to a working setup, one calm step at a time. Nothing here assumes you've done it before.

5 min read no setup yet just reading

Let's start with the worry most people have, said out loud: "I'm not technical enough for this." Here's the honest answer — you don't have to be. Hacky Hours was built, in its own words, for "everyone, from first-time builders to seasoned engineers." Your job isn't to write code. Your job is to know what you want and make decisions. The tools do the typing.

What this module covers 🌱 Five short lessons: (1) this reassurance, (2) what you'll need and what it costs — honestly, (3) what a terminal is and how to open it, (4) installing Claude Code, and (5) installing the Hacky Hours skill. At the end you'll run your first command. That's the whole climb.

A few words, demystified

You'll meet some jargon over the next few lessons. None of it is as scary as it sounds. Here's the plain-language version of the words that matter most:

  • Code — instructions a computer follows. An app (or "application") is a bundle of that code doing something useful — a website, a tool, a game.
  • Claude — the AI assistant you may already have chatted with at claude.ai.
  • Claude Code — a version of Claude that can actually build software on your computer, not just talk about it. This is what Hacky Hours runs inside.
  • A skill — an add-on that teaches Claude Code a specific way of working. /hacky-hours is one such skill. You'll install it in Lesson 5.
  • The terminal — a plain text window where you type commands instead of clicking buttons. You'll use it just a little, and Lesson 3 makes friends with it.

You drive the product. Claude helps you build it — but it needs your direction.

That line is the heart of Hacky Hours, and it's good news for a non-coder: the most important thing you bring isn't technical skill, it's knowing what you're trying to make and caring about whether it's any good.

An honest word before we spend any money

This part isn't free, and we won't pretend it is This course is free. But using /hacky-hours needs Claude Code, and Claude Code needs a paid Claude plan (or pay-as-you-go credits). The next lesson lays out every cost plainly so there are no surprises — and if the price isn't right for you yet, that's completely okay. You can read this whole course for free and set things up later.
