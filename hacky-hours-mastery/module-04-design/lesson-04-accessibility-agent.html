The accessibility agent

Accessibility means making what you build usable by people with disabilities — and it's not an extra. At Empathetech it's the whole point. This lesson introduces the agent who keeps it central from day one, and is honest about what she can and can't promise.

Why this is central at Empathetech

Empathetech builds software designed to be usable by as many people as possible — accessibility is its mission, not a feature it bolts on. So in Hacky Hours, accessibility isn't something you have to remember to ask for. It's a default the team brings to you.

From the Hacky Hours values "Accessible by default. WCAG 2.1 AA is the right starting point for any UI. Don't wait for the user to ask. Semantic HTML before complex UI frameworks."

WCAG 2.1 AA: the starting target

WCAG stands for the Web Content Accessibility Guidelines — the widely-used international standard for making web content accessible, published by the W3C (the group that sets web standards). It comes in levels: A, AA, and AAA. AA is the level most organizations aim for, and it's the one Hacky Hours uses as a starting point.

It covers practical things: enough color contrast between text and background so people with low vision can read it; everything reachable with a keyboard for people who can't use a mouse; and content that works with a screen reader — a tool that reads the page aloud for people who are blind.

What "accessible by default" looks like

One phrase from the values does a lot of work: "Semantic HTML before complex UI frameworks." Semantic HTML means using the web's built-in elements for what they actually are — a real button for a button, a real heading for a heading — instead of faking them with generic boxes.

This matters because screen readers and keyboards already understand the real elements. A genuine <button> can be focused, pressed with the keyboard, and announced as "button" automatically. A <div> styled to look like a button does none of that unless someone rebuilds it by hand — and often no one does.

The most accessible code is usually the most ordinary code, used honestly.

Try it: which alt text is right?

Alt text is a short written description of an image that a screen reader reads aloud, so someone who can't see the image still gets its meaning. Good alt text describes the image's role in the page — not every visual detail.

What the agent does and doesn't promise

The honest limit Lena helps you build accessibly and aim at WCAG 2.1 AA — that's real and valuable. But the agent does not certify legal compliance, and neither does this lesson. Meeting an accessibility standard is something assessed by real testing — including by people who use assistive technology — not something a document can guarantee. If you're in a setting with legal accessibility requirements, the responsibility to verify them stays with you.

That limit doesn't make the agent less useful — it makes it honest. Building accessibly from day one means there's far less to fix later, and the people you serve are included from the start. Verification is the next step, not a replacement for the head start.

Learning more on your own 🌱

Accessibility is one of the most rewarding crafts to grow into — every bit you learn directly includes more people.

Good next steps
  • Read the W3C's Introduction to Web Accessibility — the clearest plain-language starting point there is.
  • Try navigating a website you know using only the Tab key. You'll feel instantly what keyboard accessibility means.
  • Read how the agent thinks: /hacky-hours team show lena --prompt.
