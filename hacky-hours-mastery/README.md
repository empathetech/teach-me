# Hacky Hours Mastery

A hands-on course that teaches you to use the **`/hacky-hours`** skill (v4.0.1) confidently —
and to grow into a real builder along the way, not just someone aimlessly prompting an LLM.

> 🎯 **Goal:** Use `/hacky-hours` confidently to build real, shippable software — and treat its
> stakeholder agents as people you learn from, so you grow steadily in security, accessibility,
> data, and deployment.

An **Empathetech** community course. Built for people who are *not* professional software
engineers but want to build real software with care.

## How to use this course

1. Open **`dashboard.html`** in your browser (just double-click it).
2. Work through the modules in order, ~30 minutes a day. No rush.
3. Many lessons ask you to run the real `/hacky-hours` skill in Claude Code and reflect — that's
   where the learning sticks. ([Install Hacky Hours](https://github.com/empathetech/hacky-hours-docs).)
4. Come back any time and open this folder in Claude Code — your tutor will pick up where you
   left off.

## An honest note on what this teaches

Hacky Hours — and this course — give you **direction, not guarantees**. Especially around
security and privacy: the tools help you avoid common mistakes and ask better questions, but the
responsibility to review, understand, and verify what you build stays with you. For anything that
handles real people's sensitive data, get a qualified review. Knowing where your own knowledge
ends is itself a skill this course wants to build.
